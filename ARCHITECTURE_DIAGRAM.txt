║                          CALL EVENT TRACKING                                 ║
╚══════════════════════════════════════════════════════════════════════════════╝

Call arrives → Newchannel event → Creates call record in DB
     │
     ▼
Call rings → Dial event → Updates status to 'ringing'
     │
     ▼
Call answered → Answer event → Updates status to 'answered'
     │
     ▼
Calls connected → Bridge event → Updates status to 'connected'
     │
     ▼
IVR option selected → VarSet event → Stores IVR selection
     │
     ▼
Call ends → Hangup event → Calculates duration & revenue
     │
     ▼
Stored in MongoDB → Available in dashboard & API
